## Objetivo
Ao criar uma nova temporada após encerrar a anterior, herdar automaticamente todas as configurações da última temporada (pontuação, prêmios, blinds, financeiro, regras, eliminações, frequência) em vez de cair nos valores padrão fixos. O usuário só precisa ajustar o que mudar (nome, datas).

## Comportamento esperado
- **Nova temporada sem histórico** (primeiro uso): mantém os defaults atuais.
- **Nova temporada com pelo menos uma temporada anterior**: pré-preenche o formulário com os dados da temporada mais recente (a última encerrada, ou a mais recente por `createdAt`).
- Campos que NÃO são herdados (sempre começam zerados/limpos):
  - `name` (vazio para o usuário definir)
  - `startDate` (hoje)
  - `expectedEndDate` (vazio)
  - `jackpot` e `clubFund` (controlados pelo encerramento, não pelo formulário)
  - `hostSchedule` (vazio — datas da temporada anterior não fazem sentido)
- Campos herdados:
  - `gameFrequency`, `gamesPerPeriod`
  - `scoreSchema`
  - `weeklyPrizeSchema`, `seasonPrizeSchema`
  - `blindStructure` (com novos `id`s via uuid para evitar colisão)
  - `financialParams` (buyIn, rebuy, addon, jackpotContribution, clubFundContribution, pixKey)
  - `houseRules`
  - `eliminationRewardConfig`

## Alterações técnicas
1. **`src/pages/SeasonConfig.tsx`**
   - Obter `seasons` via `usePoker()`.
   - Calcular `previousSeason` = temporada mais recente que não seja a `activeSeason` (ordenar por `endDate ?? createdAt` desc). Passar para `useSeasonForm`.

2. **`src/hooks/useSeasonForm.ts`**
   - Aceitar novo parâmetro opcional `previousSeason: Season | null` e repassar para `useSeasonFormInitializer`.

3. **`src/hooks/season/useSeasonFormInitializer.ts`**
   - No branch `isCreating && !activeSeason` (ou quando `activeSeason` é null), se `previousSeason` existir, usar seus valores como base ao invés dos defaults hardcoded:
     - `setValue('gameFrequency', previousSeason.gameFrequency)`, idem `gamesPerPeriod`, `buyIn`, `rebuy`, `addon`, `jackpotContribution`, `clubFundContribution`, `pixKey`, `houseRules`.
     - `setScoreEntries([...previousSeason.scoreSchema])`, idem prêmios semanais/finais.
     - `setBlindLevels(previousSeason.blindStructure.map(l => ({ ...l, id: uuidv4() })))`.
     - `setValue('eliminationReward*')` a partir de `previousSeason.eliminationRewardConfig` (fallback para desabilitado quando ausente).
     - `setHostSchedule?.([])` — não copiar agenda.
     - `name` continua vazio; `startDate` = hoje; `expectedEndDate` = vazio.
   - Se não houver `previousSeason`, manter exatamente os defaults atuais (sem regressão).

4. **Indicador visual (pequeno)** em `SeasonConfig.tsx`: quando o formulário é pré-preenchido a partir da anterior, exibir um aviso discreto no topo ("Configurações herdadas da temporada anterior. Ajuste o que precisar.") com um botão "Limpar e usar padrões" que reseta para os defaults atuais.

## Garantias de não quebra
- Nenhuma alteração de schema, repositório, ou lógica de encerramento/jackpot/ranking.
- Defaults antigos preservados para o cenário sem histórico.
- IDs de blinds regenerados, evitando conflito com a temporada anterior no banco.
- `hostSchedule` continua sendo gerado depois pelo `AutoGenerateDialog` (já integrado com `expectedEndDate`).

## O que será entregue
- Pré-preenchimento automático ao criar nova temporada após encerramento.
- Aviso visual + botão para voltar aos padrões caso o usuário prefira começar do zero.
